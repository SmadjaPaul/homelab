# Recommandations d'utilisation

Quelques bonnes pratiques pour bien utiliser la plateforme.

## Connexion et compte

- **Un seul compte** — Utilisez le même compte (auth.smadja.dev) partout où c’est demandé.
- **Mot de passe** — Choisissez un mot de passe solide et ne le partagez pas.
- **Déconnexion** — Sur un ordinateur partagé, pensez à vous déconnecter après utilisation.

## En cas de problème

1. **Vérifier la page Status** — [status.smadja.dev](https://status.smadja.dev) indique si un incident est en cours.
2. **Réessayer plus tard** — En cas de panne côté plateforme, on fait le maximum pour rétablir le service rapidement.
3. **Signaler via Feedback** — Pour un bug ou une idée : [feedback.smadja.dev](https://feedback.smadja.dev). Précisez le service concerné et ce qui s’est passé.

## Données et stockage

- Les **limites de stockage** (par service ou globale) sont décrites dans la page [Limites et quotas](/user-guide/limits).
- Évitez de stocker des fichiers inutiles ou des doublons pour ne pas saturer l’espace.
- Pour les contenus sensibles, respectez les usages prévus (pas de partage non autorisé, etc.).

## Performances

- Connexion internet stable recommandée pour les applications web.
- Si une page est lente, réessayer après quelques minutes ; en cas de lenteur persistante, signaler via Feedback.

## Confidentialité et sécurité

- Ne partagez pas vos identifiants.
- Déconnectez-vous sur les postes publics ou partagés.
- En cas de doute sur un lien ou un email, vérifier l’adresse (smadja.dev) ou nous contacter.
